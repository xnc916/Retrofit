简述

GitHub API是标准的RESTful架构，通过GitHub API，你能实现几乎所有的GitHub功能。其官方文档为GitHub API v3。

以下的内容来自其官方文档，接口中省略了一些我们不需要的参数。

另外，凡是没有写完整的url路径的地方，都是以https://api.github.com为URL端点。

OAuth流程

1. 重定向用户到GitHub登录页面

GET https://github.com/login/oauth/authorize

参数

名称 类型 描述
client_id string 必要。 这是当你在GitHub上注册App时获得的Client Id。
scope string 一个可访问域的列表。如果不提供，默认是空列表，代表用户没有给应用授权任何权限。

我们应用中使用的scope包括：

  1. user : 读写用户信息；
  2. public_repo : 该用户公有库的访问权限，关注(starring)其他公有库的权限;
  3. repo : 该用户公有和私有库的访问权限。

2. 获取访问令牌

如果用户接受你的请求，GitHub会重定向到你的网址，并使用code参数给你一个临时授权码。

然后，你要使用此授权码换取访问令牌(access token):

POST https://github.com/login/oauth/access_token

参数

名称 类型 描述
client_id string 必要。 这是当你在GitHub上注册App时获得的Client Id。
client_secret string 必要。 这是当你在GitHub上注册App时获得的Client secret。
code string 必要。前一步获取到的临时授权码。

响应

默认情况下，响应将是下列形式：

access_token=e72e16c7e42f292c6912e7710c838347ae178b4a&scope=user%2Cgist&token_type=bearer

但也可以通过设置Accept请求头来请求不同格式的响应：

Accept: application/json
{
    "access_token":"e72e16c7e42f292c6912e7710c838347ae178b4a", "scope":"repo,gist", 
    "token_type":"bearer"
}

Accept: application/xml
<OAuth>
  <token_type>bearer</token_type>
  <scope>repo,gist</scope>
  <access_token>e72e16c7e42f292c6912e7710c838347ae178b4a</access_token>
</OAuth>

3. 使用访问令牌来访问API

在做API访问时，使用访问令牌的方式有两种：

第一种方法是将Access Token放在查询参数中来传递：

GET https://api.github.com/user?access_token=...

更清晰的方式是将它包含在授权请求头中：

Authorization: token OAUTH-TOKEN

搜索仓库(热门仓库)

可以通过下面的接口搜索仓库，此接口除基本的三个参数外，还需要加一个分页的参数page，每页返回30条数据。

GET /search/repositories

参数

名称 类型 描述
q string 搜索关键字以及其它限定符。
sort string 排序字段，可以是stars，fortks或updated。默认：结果符合程度。
order string 排序顺序，可以是asc或desc，默认是desc。

参数q支持各种仓库搜索限定符，在我们的应用中使用的是language，即根据仓库使用的编程语言来搜索。

例子

搜索流行的使用Java语言开发的库：

https://api.github.com/search/repositories?q=language:java&page=1

获取某一仓库的README文件

下面的方法将返回对应仓库的README文件，此文件是一个Markdown格式的文件，接口返回的响应是通过base64加密后的数据，需要进行base64的解密，才可以拿到真实的响应数据：

GET /repos/{owner}/{repo}/readme

响应：

{
  "encoding": "base64",
  "content": "encoded content ..."
}

获取Markdown文档的原始文件

此API的返回响应不是JSON字符串，而是可以通过添加请求头信息text/plain，以纯文本形式接收，最终返回的结果是HTML的文本，可以直接通过WebView加载即可。

POST /markdown/raw

参数

添加请求头信息 Content-Type:text/plain接收Markdown文档，返回该文档对应的原始文件。

响应

例如：

Status: 200 OK
Content-Type: text/html
X-RateLimit-Limit: 5000
X-RateLimit-Remaining: 4999

<p>Hello world github/linguist#1 <strong>cool</strong>, and #1!</p>

搜索用户

通过各种条件查询用户，每页默认30条：

GET /search/users

参数

名称 类型 描述
q string 搜索条件。
sort string 排序字段，可以是followers，repositories或joined。默认：结果符合程度。
order string 排序顺序，可以是asc或desc，默认是desc。

例子

假设你在查询流行开发者列表，你可以试试这样查询：

https://api.github.com/search/users?q=repos:>42&followers:>1000

这里，我们是在查询仓库数大于42，follower数量大于1000的开发者。
